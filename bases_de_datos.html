Bases de Datos

Una base de datos es un almacén que nos permite guardar un conjunto de información del mismo contexto. Estas se manejan a través de Sistemas de Gestión (DBMS) que realizan almacenamiento ordenado. Una base de datos se puede clasificar según sus modelos y paradigmas, cada uno de estos posee una estructura organizacional, jerarquía y contenido.Las hojas de cálculo también son una buena forma para almacenar información, pero estas fueron creadas para un solo usuario o un pequeño grupo donde no se tengan que manejar datos muy complicados esta es la principal razón por las que se optan por bases de datos que pueden contener cantidades masivas de información de múltiples usuarios.

Según su contenido las bases de datos pueden ser:

Bibliográficas: Contiene material de lectura como libros y revistas.

De texto Completo: Contiene textos históricos o documentales.

Directorios: Contiene listas de datos personales como correos electrónicos, nombre, números de teléfono, etc.

Especializadas: Estas contienen información técnica sobre necesidades puntuales de un público.

Algunos ejemplos son guías telefónicas, archivos personales, bibliotecas públicas, registro de transacciones e historial médico.

¿Qué es MySQL?

Es un sistema de gestión relacional basado en SQL(Structured Query Language), diseñado y optimizado para aplicaciones web, a medida que han surgido diferentes requisitos con internet, MySQL se convirtió en la plataforma elegida por empresas de comercio electrónico que administran transferencias de dinero.

Las diferencias entre SQL y NoSQL son que la primera permite combinar y extraer información de forma eficiente mientras que la segunda no lo permite o de forma muy limitada. NoSQL permite un escalado horizontal sin problemas, pero SQL es mucho mas complicado al gestionar datos junto a relaciones existentes (NoSQL no maneja sistemas relacionales).

Algunos escenarios pueden ser resueltos de forma ideal solo con uno de ellos ya que son diferentes y excluyentes a continuación hablaremos de situaciones en que son útiles cada una de estas:

SQL es ideal para el desarrollo web con jerarquía de datos, análisis de negocios o software empresarial.

NoSQL es ideal para la creación de redes sociales, el desarrollo móvil y las Cloud (XaaS).
